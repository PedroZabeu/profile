# Feature Plan: F-4.2 - Tactical Archive (Adventure Sidebar)

## üèîÔ∏è Overview
This feature populates the Global Recon Grid with the 13 adventures organized into **Operational Clusters**. The sidebar transforms from the loading state to a fully functional **Tactical Archive** that serves as the primary navigation interface for the globe's orbital locking system. This feature builds upon the `/adventures` route established in F-4.1 with the Heritage Path aesthetic.

## üé® Visual Design & Theming

### Layout Structure
- **Sidebar Container**: Fixed-width panel with terminal framing (`1px solid Glacier Steel at 20% opacity`)
- **Cluster Headers**: Collapsible sections with status indicators `[ VERTICAL_OPS ]`, `[ ALPINE_OPS ]`, `[ OVERLAND_OPS ]`
- **Adventure Items**: Interactive list items with hover states and selection indicators
- **Status Bar**: Bottom panel showing system readiness and coordinate feeds

### Visual Identity
- **Typography**: Oxanium for all technical labels and cluster headers
- **Color States**:
  - **Default**: `text-glacier-steel` (#7A8FA3)
  - **Hover**: `text-alpine-blue` (#82B7F5) with subtle glow
  - **Active/Selected**: Fresh Powder (#FFFFFF) with lock indicator `[*]`
- **Animations**: Character-by-character typing for cluster expansion and system status updates

## üé¨ Global Recon Boot Sequence
The entrance to the Global Recon interface follows a high-fidelity tactical sequence designed to simulate a satellite link establishment.

### Phase 1: Grid Initialization ($T=0.0s$)
- **Frame Reveal**: The primary navigation grid (1px container borders) resolves immediately. Borders use a subtle flicker/glitch entrance to establish the "Terminal" aesthetic.
- **Atmosphere**: `TopographyBackground` and `ScanLine` overlays manifest, creating the environmental depth.

### Phase 2: Command & Control reveal ($T=0.2s - 0.8s$)
- **Header Metadata**: `[ ADVENTURES ]` and `STATUS: ONLINE` slide/fade into position.
- **Technical Readouts**: `[ GLOBAL_RECON_FEED ]` and empty coordinates `LAT: --.-- / LON: --.--` fade into the central section.
- **System Logs**: Footer identification and contact nodes resolve (Static/Fade).

### Phase 3: Tactical Archive Manifestation ($T=1.0s - 2.5s$)
The sidebar populates using a prioritized **Sequential Header Typing** sequence. No typing animations are applied to individual items or secondary data‚Äîthe focus is strictly on the Operational Clusters.
- **Step 1 ($1.0s$ - $1.5s$)**: `[ VERTICAL_OPS ]` types letter-by-letter. Upon completion, its member list manifests (fast staggered fade).
- **Step 2 ($1.6s$ - $2.1s$)**: `[ ALPINE_OPS ]` types letter-by-letter. Upon completion, its member list manifests.
- **Step 3 ($2.2s$ - $2.7s$)**: `[ OVERLAND_OPS ]` types letter-by-letter. Upon completion, its member list manifests.

### Phase 4: Full Recon Stability ($T=2.8s+$)
- **Globe Reveal**: The 3D Orbital Globe scales from `0.95` to `1.0` and fades in over 1.2s.
- **Link Stability**: System status in footer transitions to `SIGNAL_STABLE // RECON_ACTIVE`.

## üõ†Ô∏è Implementation Strategy

### 1. Unified Animation Orchestration
- Use a central `motion.div` or `AnimatePresence` to coordinate the timing between `ReconLayout` components.
- Implementation of a custom `Typewriter` component specifically for the Cluster Headers.

### 2. Interaction States
- **Initial State**: All clusters visible (no overflow/scroll needed for manifest) but items start at 0 opacity.
- **Hover State**: Subtle blue glow (`text-alpine-blue`) and cursor pointer on adventure items.
- **Selected State**: Fresh Powder (#FFFFFF) text with lock indicator `[*]`, triggers globe rotation lock.

### 3. Globe Integration
- **Selection Handler**: Adventure click triggers immediate coordinate update in the `RECON_FEED`.
- **Coordinate Mapping**: Convert adventure LAT/LON to COBE rotation angles.
- **Lock Sequence**: Visual feedback `[ TARGET_ACQUIRED ]` in footer when rotation completes.

### 4. Mobile Adaptations
- **Drawer State**: Toggle between collapsed/expanded mobile drawer
- **Touch Gestures**: Swipe gestures for drawer control
- **Selection Optimization**: Larger touch targets for mobile interaction
