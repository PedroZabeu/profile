This feature implements the **Adventure Briefing Terminal**, a high-fidelity modal interface that allows users to explore specific adventures. Unlike previous manual triggers, this modal now opens **automatically** after a successful location lock-on, creating a fluid transition between global exploration and deep-dive discovery.

## ğŸ¯ Core Interaction Mechanics

### Trigger Mechanism (The Auto-Deep Dive)
To create a seamless "satellite-to-ground" experience, the modal follows this automated lifecycle:
1. **Selection**: User clicks an adventure in the sidebar.
2. **Orbital Positioning**: Globe rotates and locks (F-4.3). The system status flashes `[ POSITIONING... ]`.
3. **Intel Delay**: After the globe reaches the location coordinates (`dist < 0.01`), a **0.5s technical delay** is applied.
4. **UX Feedback**: During this delay, the sidebar item or a central reticle flashes `[ DOWNLOADING_LOGS... ]`.
5. **Auto-Manifest**: The Briefing Modal opens automatically.

## ğŸ› ï¸ Implementation Strategy

### 1. Auto-Trigger Logic
- Integrate `useEffect` in `AdventuresPage` that monitors the distance to target (calculated in F-4.3).
- When `dist < 0.01`, trigger a 500ms `setTimeout`.
- Show temporary UI feedback `[ DOWNLOADING_LOGS... ]` in the system log footer during this wait.
- Automatically set `isModalOpen(true)` after timeout.

### 2. Animations (EPIC 3 Style)
- Use `AnimatePresence` for the modal.
- `boxVariants`: `initial: { scale: 0.9, opacity: 0 }`, `animate: { scale: 1, opacity: 1 }`, `exit: { scale: 0.9, opacity: 0 }`.
- Ease: `easeOut` with `0.4s` duration.

### 3. Visual Assets (PLACEHOLDER PHASE)
- **Requirement: Square 1:1 container**.
- Use the high-tech placeholder container defined in the previous plan.
